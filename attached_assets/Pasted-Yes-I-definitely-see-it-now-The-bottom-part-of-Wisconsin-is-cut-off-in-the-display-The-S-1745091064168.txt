### 🔍 Root Cause

Your current code only adds a `viewBox` to national maps (`presidential.svg`, `states.svg`). But for **state-level maps like `wi.svg`**, you're leaving the SVG untouched:

```python
if "presidential" in svg_path or "states" in svg_path:
```

If the original SVG **doesn’t include a `viewBox`**, and you don’t inject one, **browsers can't scale it properly**, especially in responsive containers.

### ✅ The Fix

You need to dynamically extract or estimate and inject a `viewBox` into all SVGs — **not just presidential and state maps**.

```python
if 'viewBox=' not in svg_data:
```

With this more robust fix:
```python
if 'viewBox=' not in svg_data:
    # Try to extract width/height to estimate viewBox
    width_match = re.search(r'width="(\d+)"', svg_data)
    height_match = re.search(r'height="(\d+)"', svg_data)

    width = int(width_match.group(1)) if width_match else 1000
    height = int(height_match.group(1)) if height_match else 600

    svg_display = re.sub(r'<svg', f'<svg viewBox="0 0 {width} {height}"', svg_data)
else:
    svg_display = svg_data
```

---

### 💬 Why This Works
This ensures that **every SVG** has a proper `viewBox`, which is the **only way** modern browsers can scale SVGs inside divs responsively. Without it, **bottom padding gets cut off** exactly like in your screenshot.
